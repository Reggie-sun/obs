- 部署更轻。没有额外词库和分词依赖，直接本地可跑。

- 代价也明显。它不懂真实词边界，会产生跨词噪声，精度不如 jieba。

    这里主分词默认是 jieba_search，配置在 config.py (line 64)，补充 2-gram 权重默认 0.35，配置在 config.py (line 65)。
    
- 向量分支和词法分支之间：  
    在 retrieval_service.py (line 145) 到 retrieval_service.py (line 194)，不是 a * vector_score + b * lexical_score，而是 RRF：  
    fused += 1 / (rrf_k + rank)  
    默认 rrf_k = 60，见 config.py (line 63)。
    

所以当前真实公式可以理解成：

`lexical_score = bm25(jieba主分词) + 0.35 * bm25(补充2gram) final_fused_score = 1 / (60 + vector_rank) + 1 / (60 + lexical_rank)`

如果一个 chunk 只出现在一条分支，就只吃那一条的 RRF 分；如果两条都命中，就两边相加。再往后如果你走 chat 的 heuristic rerank，那是另一层 0.7 * 检索分 + 0.3 * token overlap，不要和 hybrid 召回层混在一起。
