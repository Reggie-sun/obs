## 二、故障经过

### 2.1 报警与初步响应

**10:23** — 操作员王师傅报告A线3号机器人在执行第4工位工件抓取时突然停止，HMI显示E203碰撞报警。王师傅说他看到机器人大臂蹭到了夹具边缘，工件（铝合金件，大约2.5kg）掉落在定位台上，没砸到人。

**10:25** — 设备工程师张工到达现场（从B线跑过来的，大概2分钟）。孙工（电气工程师）同步收到DFMS报警推送，电话确认现场无人受伤后开始远程调取机器人控制器日志。

初步查看：
- J2轴外壳上有轻微接触痕迹（蹭掉了一点点漆）
- 夹爪右侧指有轻微划痕但不影响功能
- 没有看到明显的变形或裂纹
- 掉落工件位于定位台右侧约5cm处，未弹出安全围栏区域
- 附近无人员活动，无安全隐患

### 2.2 第一轮排查（机器人本体）

**10:30-11:00** — 第一轮排查（聚焦机器人本体）：
- 切换手动模式，低速点动各轴 — 动作顺畅无卡阻
- 检查各轴外观 — 无变形、裂纹
- 检查减速器状态 — 各轴无异响，温度正常（J1-J6轴温度均在35-42°C范围内）
- 检查夹爪 — 右侧有划痕但功能正常，气缸动作正常
- 检查线缆 — 各轴线缆无磨损、无干涉
- 执行回零 — 各轴回零正常，无偏差
- 调取控制器力矩日志 — J2轴在碰撞瞬间力矩峰值达到额定值的127%（E203触发阈值120%），确认碰撞力不大
- 复位报警，清除故障码

### 2.3 试运行与二次碰撞

**11:00-11:30** — 试运行（逐步加载验证）：
- 空载低速1循环 → 正常
- 空载正常速度1循环 → 正常
- 上料负载运行 → **又撞了**。E203再次触发，同一工位同一位置。

**11:32** — 王师傅报告：第二次碰撞位置和第一次几乎一样，都是4号工位抓取上升阶段，大臂蹭到夹具右侧边缘。

> 当时就觉得不是偶然碰撞了，空载没事负载就撞，十有八九是工件定位问题。负载后工件重心和空载时夹爪不一样，导致抓取轨迹有微小偏差。而偏差恰好在这个工位够到了夹具边缘。——张工

### 2.4 第二轮排查（夹具与定位系统）

**11:35-12:30** — 第二轮排查（重点查夹具和定位）：
- 逐工位检查定位夹具
- **发现4号工位定位销可以用手晃动**（不应该能晃动，正常状态应完全固定）
- 千分表测量：定位销偏移约0.3mm（X方向+0.22mm，Y方向-0.18mm）
- 拆下来看：底座螺纹磨损严重，螺纹配合间隙明显增大
- 查阅领用记录：这个定位销从2022年9月安装使用，已约3年
- 其他工位也查了一遍：
  - 1号工位：偏移约0.08mm，螺纹轻微磨损
  - 3号工位：偏移约0.12mm，螺纹轻微磨损
  - 2、5、6号工位：正常，偏移<0.03mm
- 比对机器人示教轨迹：原始轨迹与夹具边缘的安全间距设计值为5mm，考虑0.3mm偏移后实际最小间距降至约2mm，叠加负载变形（约1mm）和工件定位公差（±0.5mm），安全间隙已不足

**故障机理示意**：
```
正常状态:                    故障状态:
┌──────┐                    ┌──────┐
│ 工件  │ ←5mm安全距→ │夹具边│
│(定位准)│                    │      │
└──┬───┘                    └──────┘
   │                              ↑ 仅约2mm
定位销固定                   ┌──────┐
                              │ 工件  │~~~→│夹具边│
                              │(偏0.3)│    │ 碰撞!│
                              └──┬───┘    └──────┘
                                 │
                              定位销松动偏移
```

*（这里浪费了一些时间。一开始还是习惯性地先查机器人本体，查了半小时才想到可能是夹具问题。以后碰到碰撞报警应同时检查夹具和定位系统，不应只盯着机器人本体。——张工自我反思）*

### 2.5 维修实施

**12:30-13:00** — 维修：
- 更换4号定位销（新件库位B区07货架，规格M8x30，材质：淬火合金钢）
- 顺便把1号和3号定位销也紧固了（螺纹尚可，添加螺纹紧固胶Loctite 243）
- 紧固扭矩：25N·m（按厂商手册要求）
- 千分表验证：
  - 4号工位：定位精度恢复到0.02mm以内
  - 1号、3号工位：定位精度恢复到0.03mm以内
- 全部6个工位定位精度复检一遍，均在0.03mm以内合格
- 记录更换信息：旧件报废处理，新件批次号记录至DFMS

### 2.6 验证恢复

**13:00-13:30** — 验证（逐步恢复生产流程）：
- 空载低速3循环 → OK
- 负载正常速度3循环 → OK
- 负载高速（生产节拍速度）5循环 → OK
- 连续跑30分钟 → 无报警，生产约150件产品
- 首件检验合格（质量部QC确认尺寸、外观均OK）
- 通知生产计划部：A线恢复，当日产量缺口通过加班补回

**13:47** — 确认恢复，正式投入生产。

## 三、根本原因分析

### 3.1 5 Why 分析

1. 为什么碰撞？→ 机器人轨迹与夹具干涉
2. 为什么干涉？→ 工件位置偏了约3mm（定位销偏移0.3mm，叠加负载变形和公差）
3. 为什么偏？→ 4号定位销松动
4. 为什么松动？→ 底座螺纹磨损，3年未更换
5. 为什么3年没换？→ **没有定位销的定期更换计划，日常点检未涵盖此项检查**

### 3.2 原因分类

| 类别 | 原因 | 影响程度 |
|---|---|---|
| **直接原因** | 4号定位销松动（偏移0.3mm） | ★★★★ |
| **根本原因** | 定位销无预防性更换机制，日常点检缺失此项 | ★★★★★ |
| **促成因素** | 前两次E203报警未深入排查根因 | ★★★ |
| **管理原因** | 厂商操作手册中定位销更换周期建议未被识别和纳入维护体系 | ★★★ |
| **人员因素** | 碰撞排查时过度聚焦机器人本体，未同步检查夹具定位 | ★★ |

### 3.3 厂商技术支持反馈

> 厂商技术支持（王工，9/16邮件回复）确认：
> - 定位销建议更换周期为2年（参考操作手册第7章第7.3节）
> - 如使用环境存在持续振动或冲击，建议缩短至18个月
> - 建议每季度检查一次定位销紧固状态（点检项）
> - 我们3年未更换已超出建议周期50%，磨损程度符合预期
> - 推荐使用Loctite 243中强度螺纹紧固胶作为辅助固定

## 六、经验教训

### 6.1 技术层面

1. **碰撞报警排查应同时检查夹具定位**，不要只盯着机器人本体看。空载正常、负载就撞，大概率是工件定位偏了。建议排查顺序：复位→空载测试→负载测试→若负载复现则立即查夹具定位。

2. **低成本易损件也要有预防性更换计划**。定位销这种几块钱的东西，不换造成的停机损失是它本身价格的数百倍。

3. **重复报警就是预警信号**。前两次E203（8/28、9/5）就是定位销逐渐松动的早期征兆，当时复位了事没有深究，错过了早期干预的机会。

4. **力矩日志是重要诊断依据**。本次通过控制器力矩日志确认碰撞力为额定127%（不大），帮助快速排除了机器人本体损伤的可能性。

### 6.2 管理层面

5. **厂商操作手册中的保养建议必须梳理纳入维护计划**。不能等设备到货了就把手册束之高阁，新设备验收时就应该把所有易损件更换周期提取出来建台账。

6. **点检项目需要持续优化**。现有日常点检表没有覆盖定位销紧固检查，说明点检项目的设计和更新存在盲区。

7. **同类问题横向排查应制度化**。一个问题工位发现了，要主动扩展到全线甚至全厂同类设备，不能头痛医头。

### 6.3 流程改进建议

- 碰撞报警SOP增加"夹具定位同步检查"步骤（已修订，见CAPA-8）
- 建立**易损件台账**，记录安装日期、更换周期、库存情况
- 对重复性报警（同一代码3次/月）自动触发RCA要求
