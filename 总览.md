
1. **数据层 (Ingestion)** — `DocumentIngestionService` 把原始文件通过 解析→OCR→筛选→清理→分块→嵌入→写入Qdrant 的完整链路变成可检索的向量数据
2. **索引层 (Indexing)** — `QdrantVectorStore` 封装 Qdrant 的读写操作，支持本地/内存/远程三种部署模式，payload 携带 30+ 元数据字段
3. **检索层 (Retrieval)** — `RetrievalService` 执行 向量召回+词法召回→RRF融合→OCR治理→权限过滤→Rerank 的多路检索，支持部门优先和补充召回
4. **生成层 (Generation)** — `ChatService` 完成 查询改写→检索引用→Prompt裁剪→LLM生成 的端到端问答，包含降级兜底和外部知识回退机制
5. **评估体系（eval）— 1. **V1 离线评测** (`backend/eval/`) — 40 条最小基准样本，mock embedding + 内存 Qdrant 跑，测 Hit@1/3/5、MRR、部门优先命中率、拒答正确率
    
6. **V2 在线评测** (`scripts/eval_retrieval.py`) — 调真实 `/retrieval/search` 端点，支持部门身份登录、阈值实验矩阵、retrieval_context 快照记录，核心指标是 top1_accuracy / topk_recall / supplemental_precision / supplemental_recall
    

**CI 门禁**：`check_eval_thresholds.py` 检查评测结果是否满足最低阈值（默认 top1 ≥ 0.95, topk ≥ 0.95）

**结果对比**：`compare_eval_results.py` 做前后评测差异分析，检测回归

**阈值实验矩阵**：5 组预定义实验配置（current_defaults / lower / aggressive / conservative / stricter_fine），结果输出到 `eval/experiments/`
