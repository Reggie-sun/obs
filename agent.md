什么时候适合agent

适合 Agent 的场景是“规则难写、目标变化快、需要多步探索”：

1. 需求不确定、要边试边改
    - 例如“帮我找到更高通过率的提示词策略”。
2. 任务需要跨系统编排
    - 查日志、改配置、跑实验、回收结果、再决策。
3. 评价标准主观或复杂
    - 如“回答更自然、更像人工客服”，难用单一规则判定。
4. 长链路自动化运维

- 训练失败自动诊断并重试不同参数。

不太适合 Agent 的场景：

1. 有硬约束和可量化验收（你现在这个就是）
2. 要强可复现、可审计
3. 成本/延迟敏感

一句话：

- **确定性流水线** → 脚本/规则优先
- **探索性决策流程** → Agent 更合适
