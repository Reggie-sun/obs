# 1. 目的

为规范设备报警（Alarm）处理流程，提升设备故障响应效率，降低停机时间（Downtime），避免误操作带来的安全风险及设备损坏，特制定本标准作业程序。

---

# 2. 适用范围

本SOP适用于：

- 自动化产线设备报警处理
- 设备运行异常排查
- 设备维护人员与操作人员协同处理流程
- 厂区内所有生产设备的日常巡检与预防性维护
- PLC控制系统、SCADA系统、MES系统的报警联动处理

【注意：关于CNC加工中心的报警处理，不在本SOP范围内，请参见SOP-CNC-ALARM-002。但由于历史原因部分CNC报警代码与自动化设备重叠（如E101、E305），查询时请注意区分设备类型。】

# 3. 术语定义

| 术语 | 定义 |
|---|---|
| Alarm Code | 设备报警代码，由设备厂商定义，通常由字母前缀+数字组成 |
| Critical Alarm | 严重报警，设备立即停止运行，需人工干预方可恢复 |
| Warning Alarm | 警告报警，设备继续运行但需关注，可能升级为Critical |
| Info Alarm | 信息提示，仅记录日志，不影响设备运行 |
| Reset | 报警复位操作，将设备从报警状态恢复到就绪状态 |
| MTTR | 平均修复时间（Mean Time To Repair），衡量维护效率的核心指标 |
| MTBF | 平均故障间隔时间（Mean Time Between Failures），衡量设备可靠性的核心指标 |
| MTTF | 平均失效时间（Mean Time To Failure），不可修复设备从开始使用到失效的平均时间 |
| OEE | 设备综合效率（Overall Equipment Effectiveness）= 可用率 x 性能率 x 良品率 |
| Root Cause | 根本原因分析，通过系统性方法找出故障的根本驱动因素 |
| Lockout/Tagout (LOTO) | 上锁挂牌程序，确保设备在维护期间不会被意外启动的安全规程 |
| PPE | 个人防护装备（Personal Protective Equipment），包括安全帽、防护眼镜、防护手套、安全鞋等 |
| PLC | 可编程逻辑控制器（Programmable Logic Controller），工业自动化的核心控制单元 |
| SCADA | 数据采集与监控系统（Supervisory Control and Data Acquisition） |
| MES | 制造执行系统（Manufacturing Execution System），连接计划层与控制层 |
| DFMS | 设备故障管理系统（Device Fault Management System），公司内部故障记录与分析平台 |

*注：以下为部分同事习惯使用的非标准叫法，仅供沟通参考，正式文档请使用上方标准术语：*
- *Alarm Code → 有同事叫"故障码"、"error code"、"错误号"*
- *Critical Alarm → 也有人叫"一级报警"、"红色报警"、"急停类报警"*
- *Reset → 口语叫"消警"、"复位"、"清报警"、"reset一下"*
- *LOTO → 安全部门有时叫"上锁挂牌"，车间叫"挂牌子"*
- *Root Cause → 有人叫"根因"，有人叫"5个为什么"，还有人直接说"RC"*

# 4. 职责

## 4.1 操作员（Operator）

- 负责第一时间响应设备报警
- 按照标准流程执行基础排查（Level 1排查）
- 无法处理时及时上报至设备工程师
- 每班开始前执行设备点检，确认设备处于正常状态
- 如实填写交接班记录，包括设备运行状况和已处理报警
- 严禁在未授权情况下修改设备参数或程序

（王师傅提醒：新来的操作员小刘上次把E101当E102处理了，浪费半小时。建议在HMI报警页面加上报警代码的颜色区分——Critical红色、Warning黄色。这个需求已经提给IT部门了，还没排期。）

## 4.2 设备工程师（Engineer）

- 负责故障深度分析与处理（Level 2/3排查）
- 执行维修或部件更换，确保修复质量
- 记录并归档故障信息至DFMS系统
- 编制和更新设备维护作业指导书
- 参与报警处理流程的持续改进
- 每月对高频报警进行统计分析，提出改善建议
- 负责备件管理和更换记录

## 4.3 生产主管（Supervisor）

- 负责异常升级处理的最终决策
- 协调资源恢复生产，包括人员调配和排程调整
- 审批超过2小时的停机维修
- 定期审核报警处理记录，评估团队响应效率

## 4.4 设备经理（Equipment Manager）

- 审批涉及设备改造或大修的方案
- 审核月度/季度设备运行报告
- 推动预防性维护计划的制定与执行
- 批准设备报废或更新换代申请

# 5. 作业流程

## 5.1 报警触发与初步处理

当设备出现报警时，应按照以下步骤执行：

1. 确认报警类型（Critical / Warning / Info）
2. 记录报警代码及发生时间（精确到秒）
3. 通知周边操作人员注意安全
4. 检查基础状态：
    - 急停按钮是否触发
    - 电源是否正常（三相电压380V±10%）
    - 气源压力是否正常（0.5–0.7 MPa）
    - 液压系统压力是否正常（参照设备铭牌参数）
    - 冷却水流量是否正常

> ⚠️ 在未确认设备安全状态前，禁止执行复位操作。

**关于"直接复位"的补充说明（2025年11月安全事件后增加）：**

去年11月B线发生过一次操作员未排查就直接reset导致二次碰撞的事故（机器人夹爪撞坏，停机8小时，损失约3.2万）。事后分析是操作员觉得"这种报警经常复位的就好了"所以没按流程检查。**请所有人员务必严格遵守先检查后复位的原则，不要凭经验跳步。**

### 5.1.1 操作员Level 1排查清单

操作员在接到报警后5分钟内应完成以下检查：

- [ ] 确认报警代码并记录
- [ ] 检查HMI（人机界面）上的报警详情页面
- [ ] 确认设备当前状态（运行/停止/急停）
- [ ] 检查周边环境是否存在安全隐患（漏水、异味、异响）
- [ ] 检查电源指示灯状态
- [ ] 检查气源/液压压力表读数
- [ ] 检查设备温度是否异常
- [ ] 如上述检查均正常，尝试一次复位操作
- [ ] 如复位失败，立即上报设备工程师

## 5.2 标准报警处理流程

### 5.2.1 气压不足报警（Alarm Code: E101）

**严重等级：** Critical（设备停机）

**现象描述：**
设备无法启动或运行中断，HMI提示"气压不足"或"Air Pressure Low"，气压表读数低于0.5 MPa。可能伴随气缸动作缓慢、夹具夹持力不足等现象。

**可能原因：**
- 空压机故障或停机
- 气路管道破裂或接头松脱
- 气源过滤减压阀堵塞
- 电磁阀故障导致持续排气
- 气缸密封件老化泄漏

**处理步骤：**

1. 检查气源总阀是否开启，确认空压机运行正常
2. 检查气压表读数是否在0.5–0.7 MPa范围内
3. 沿气路逐段检查，查找泄漏点（听声法、肥皂水检测法）
4. 检查过滤减压阀是否堵塞，必要时清洗或更换滤芯
5. 检查电磁阀动作是否正常，有无卡滞现象
6. 恢复气压至正常范围后执行复位操作
7. 空载试运行3个循环，确认设备恢复正常

**预防措施：**
- 每周检查一次气路接头紧固状态
- 每月更换一次过滤减压阀滤芯
- 每季度检查气缸密封件磨损情况

**所需工具：** 气压表、肥皂水、扳手套装、密封胶带

**预计修复时间：** 15–45分钟

*[老陈的经验：气压不足报警90%是滤芯堵了，先检查过滤减压阀能省不少时间。但去年换了新品牌的滤芯后堵塞概率明显下降，所以现在先查气路接头更稳妥。——张工]*

---

### 5.2.2 机器人碰撞报警（Alarm Code: E203）

**严重等级：** Critical（设备停机）

**现象描述：**
机器人停止运行，示教器（Teach Pendant）显示碰撞报警，可能伴随异常声响。机器人可能偏离正常轨迹。

**可能原因：**
- 工件定位偏移导致机器人路径干涉
- 夹具松动或工件脱落
- 机器人TCP（工具中心点）标定偏差
- 外部轴与机器人运动干涉
- 编程错误导致运动轨迹异常

**处理步骤：**

1. 立即确认周边人员安全，设立警示标识
2. 将设备切换至手动模式（Manual / Teach模式）
3. 在手动模式下缓慢移动机器人，检查碰撞点及周边环境
4. 检查机器人各轴有无机械损伤（变形、裂纹、异响）
5. 检查末端执行器（夹爪/焊枪等）是否受损
6. 检查工件定位夹具状态
7. 确认无损伤后，执行机器人回零（Home）操作
8. 低速空载运行一个完整循环，确认轨迹正常
9. 逐步恢复正常速度，观察3个完整循环
10. 确认一切正常后恢复自动运行模式

> ⚠️ 未确认安全前，禁止直接复位设备。碰撞后必须检查机器人各轴精度。

**预防措施：**
- 每日开工前校验工件定位基准
- 每周执行一次机器人零点校准
- 定期检查碰撞检测灵敏度设置

**所需工具：** 示教器、千分表、轨迹测试程序

**预计修复时间：** 20–60分钟

*[历史案例参考：2025-09-12 A线3号机器人E203碰撞，原因是4号夹具定位销松动导致工件偏移3mm。处理后增加了定位销紧固力矩检查项到日检表中。]*

### 5.2.3 温度超限报警（Alarm Code: E305）

**严重等级：** Warning（降级运行）→ Critical（持续超限则停机）

**现象描述：**
设备关键部位温度超出正常范围，HMI显示温度报警。可能导致加工精度下降、轴承过热、液压油温升高等问题。

**正常温度参考值：**
- 主轴轴承温度：< 70°C
- 液压油温度：30°C – 60°C
- 伺服电机温度：< 80°C
- 电控柜内部温度：< 40°C

（注：以上温度标准为通用参考值。部分进口设备的温度阈值可能不同，如德系K品牌主轴轴承报警阈值为65°C，日系F品牌为75°C，请以设备说明书为准。如有冲突，以设备铭牌标注为准。）

*以下为2024年旧版参数，已作废，请勿参考：*
*~旧版：主轴轴承温度 < 80°C（已更新至70°C）~*
*~旧版：电控柜内部温度 < 50°C（已更新至40°C）~*

**可能原因：**
- 冷却系统故障（冷却泵、冷却风扇）
- 冷却液不足或冷却液管路堵塞
- 环境温度过高
- 设备超负荷运行
- 轴承磨损导致摩擦发热
- 润滑脂干涸或润滑系统故障

**处理步骤：**

1. 确认报警温度值和报警部位
2. 检查冷却系统运行状态（冷却泵、风扇、水流量）
3. 检查冷却液液位，不足时补充至规定液位
4. 检查冷却液管路是否有堵塞或泄漏
5. 检查设备运行负载是否超出额定值
6. 检查环境通风条件，必要时开启辅助散热
7. 温度恢复正常后执行复位操作
8. 如温度持续超限，降低设备运行速度（降速50%运行）
9. 记录温度变化趋势，分析是否需要更换磨损部件

**预防措施：**
- 每日检查冷却液液位
- 每周清洁冷却散热器
- 每月检查轴承润滑状态
- 每季度校验温度传感器精度

**预计修复时间：** 10–30分钟（冷却液补充）/ 2–4小时（冷却系统维修）

*（赵工补充：夏天七八月份温度报警特别多，主要是车间环境温度高导致冷却效率下降。建议每年6月前提前检查冷却系统并清洁散热器。）*

### 5.2.4 伺服驱动器故障报警（Alarm Code: E402）

**严重等级：** Critical（设备停机）

**现象描述：**
伺服驱动器面板显示故障代码，对应轴停止运动。HMI显示驱动器报警或通讯异常。

**维修日志参考（A线2号机驱动器故障处理记录）：**
```
日期：2025-08-17 10:23
报警：E402，驱动器面板显示AL-09
处理人：张工
过程：断电重启无效→查编码器线正常→查机械传动发现减速器轻微卡阻→拆开发现润滑脂干结→清洗重加油→复位成功
耗时：2.5h  备注：减速器润滑问题已加到月检表
```

**常见驱动器故障代码：**

| 驱动器故障码 | 含义 | 处理方式 |
|---|---|---|
| AL-01 | 过载保护 | 检查负载，等待10分钟后重启 |
| AL-02 | 过电压 | 检查电源电压，确认制动电阻 |
| AL-03 | 欠电压 | 检查电源供电，确认接线 |
| AL-05 | 编码器异常 | 检查编码器接线，必要时更换编码器 |
| AL-09 | 位置偏差过大 | 检查机械传动系统，确认无卡阻 |
| AL-12 | 过热保护 | 等待驱动器冷却，检查散热条件 |
| AL-17 | 通讯错误 | 检查通讯线缆，确认参数设置 |

**处理步骤：**

1. 记录驱动器面板上的具体故障代码
2. 根据故障代码查阅对应处理方案（见上表）
3. 断电等待30秒后重新上电（硬复位）
4. 如硬复位后故障依旧，检查驱动器接线
5. 检查伺服电机与驱动器之间的编码器线缆
6. 检查机械传动系统是否卡阻
7. 如多次复位仍无法恢复，更换驱动器或联系厂商技术支持

**预防措施：**
- 每月检查驱动器散热风扇运行状态
- 每季度检查电气接线紧固状态
- 监控驱动器温度趋势，提前预警

**预计修复时间：** 15分钟（硬复位解决） / 2–4小时（更换驱动器）

**备件信息：**
- 常用驱动器型号：XXXX-YYYY（库存：3台，库位：A区12货架）
- 编码器线缆：型号ZZZZ（库存：5根，库位：A区15货架）
- 驱动器备件采购周期：约2周（进口件）
- *紧急情况可向C线调用备件，C线为备用产线*

### 5.2.5 PLC通讯故障报警（Alarm Code: E501）

**严重等级：** Critical（设备停机）

**现象描述：**
HMI显示与PLC通讯中断，设备无法接收控制指令。SCADA系统可能同时显示设备离线。

**微信群故障排查记录示例（2025-10-22夜班）：**
```
[22:15] 王师傅：B线4号机HMI黑屏了，PLC通讯中断
[22:16] 张工：先检查网线，上次也是网线松了
[22:18] 王师傅：网线拔插了没用
[22:20] 张工：交换机上B线那个口灯亮不亮？
[22:21] 王师傅：不亮，换了根网线还是不亮
[22:23] 张工：那可能是交换机端口坏了，把网线插到旁边空着的口试试
[22:25] 王师傅：好了 通讯恢复了。交换机端口坏了
[22:26] 张工：明天白天我换个交换机，先用着。记得报个工单
[22:26] 王师傅：好的
```
*注：以上仅为参考，正式处理仍应按照下方标准步骤执行。*

**可能原因：**
- 通讯线缆松动或损坏（以太网/RS485/Profibus）
- 网络交换机故障
- PLC硬件故障（CPU模块异常）
- 电磁干扰导致通讯不稳定
- IP地址冲突或网络配置错误

**处理步骤：**

1. 检查通讯线缆连接状态，重新插拔通讯接头
2. 检查网络交换机指示灯状态（Link灯、Activity灯）
3. 使用Ping命令测试PLC的网络连通性
4. 检查PLC的CPU模块运行指示灯（RUN/ERR）
5. 如PLC指示ERR状态，尝试重启PLC
6. 检查IP地址配置，确认无冲突
7. 检查通讯参数设置（波特率、数据位、校验位）
8. 如所有检查均正常但通讯仍无法恢复，更换通讯线缆
9. 记录故障现象和排查过程，录入DFMS系统

**预防措施：**
- 每月检查一次通讯线缆连接状态
- 确保通讯线缆远离动力线（间距≥20cm）
- 每季度备份一次PLC程序和参数

**预计修复时间：** 10–30分钟

### 5.2.6 液压系统异常报警（Alarm Code: E601）

**严重等级：** Critical（设备停机）

**现象描述：**
液压系统压力异常（过高或过低），油温超限，或液压泵异常噪音。可能影响液压夹具、液压缸等执行元件的正常工作。

**正常参数范围：**
- 系统压力：根据设备规格，通常在7–21 MPa
- 油温：30°C – 60°C
- 油位：油标中线以上

**可能原因：**
- 液压油不足或变质
- 液压泵磨损或损坏
- 溢流阀设定错误或阀芯卡滞
- 液压缸密封件泄漏（内泄或外泄）
- 管路接头松动或管路破裂
- 滤油器堵塞

**处理步骤：**

1. 确认液压系统压力表读数
2. 检查液压油箱油位，确认在正常范围内
3. 检查液压油颜色和气味（正常为清澈琥珀色，变质油发黑或有焦味）
4. 检查液压泵运行声音是否正常（不应有尖锐噪音或金属撞击声）
5. 检查管路接头是否有泄漏（外泄可见油迹）
6. 检查滤油器压差指示器
7. 根据检查结果执行对应维修操作
8. 维修完成后，低速运行液压系统5分钟排气
9. 逐步恢复正常工作压力

**预防措施：**
- 每日检查液压油油位和油温
- 每500工作小时更换液压油滤芯
- 每2000工作小时更换液压油
- 每月检查管路接头紧固状态

**安全注意事项：**
- 液压系统带压操作时，严禁拆卸管路接头
- 维修前必须泄压至0 MPa
- 液压油接触皮肤后应立即用肥皂清洗

### 5.2.7 传感器故障报警（Alarm Code: W301）

**严重等级：** Warning（降级运行）

**现象描述：**
传感器信号异常或丢失，可能导致设备误判工件位置、尺寸等。表现为检测不稳定、误触发或无信号输出。

**常见传感器类型及故障模式：**

| 传感器类型 | 常见故障 | 排查方法 |
|---|---|---|
| 光电传感器 | 镜面污染、对准偏移 | 清洁镜面，重新对准 |
| 接近传感器 | 感应距离变化、安装松动 | 调整安装距离，紧固螺丝 |
| 压力传感器 | 零点漂移、信号波动 | 校验零点，检查接线 |
| 温度传感器 | 测量偏差、断线 | 对比标准温度计，检查线缆 |
| 视觉传感器 | 图像模糊、标定丢失 | 清洁镜头，重新标定 |

**处理步骤：**

1. 确认故障传感器位置和类型（通过HMI报警信息）
2. 检查传感器外观是否有物理损伤
3. 检查传感器安装状态（是否松动、位置偏移）
4. 清洁传感器感应面/镜面
5. 检查传感器接线（接头松动、线缆破损）
6. 使用手动方式触发传感器，验证信号是否正常
7. 如传感器损坏，更换同型号传感器并重新校准
8. 更换后执行设备自检程序

**预防措施：**
- 每日清洁传感器感应面
- 每周检查传感器安装紧固状态
- 每月校准关键传感器（压力、温度类）

### 5.2.8 安全光栅报警（Alarm Code: E701）

**严重等级：** Critical（安全回路触发，设备停机）

**现象描述：**
安全光栅检测到人员或物体进入危险区域，设备立即停止。安全回路断开，设备进入安全锁定状态。

**处理步骤：**

1. 确认危险区域内无人员停留
2. 检查安全光栅是否被工件、碎屑等遮挡
3. 清理光栅通道内的障碍物
4. 检查光栅发射端和接收端对准状态
5. 确认安全光栅指示灯状态（绿色=正常，红色=遮挡）
6. 确认区域安全后，按下安全复位按钮
7. 执行设备启动前安全检查

> ⚠️ 安全光栅报警不得屏蔽或旁路。如需临时旁路维修，必须执行LOTO程序，并由设备经理书面授权。

---

### 5.2.9 急停按钮触发报警（Alarm Code: E801）

**严重等级：** Critical（设备停机）

**现象描述：**
急停按钮被按下，设备所有运动立即停止，动力电源切断。

**处理步骤：**

1. 确认触发急停的原因（人员安全、设备异常、其他紧急情况）
2. 排除触发急停的危险因素
3. 检查设备状态，确认急停未导致次生损害（如工件掉落、机械冲击）
4. 顺时针旋转急停按钮使其弹起
5. 执行系统复位操作
6. 执行设备回零（Home）操作
7. 低速试运行确认设备正常

**注意事项：**
- 急停后不得直接切换至自动模式运行
- 必须完成回零操作后方可恢复自动运行
- 每次急停触发必须记录原因和处理过程

---

### 5.2.10 电气过载/短路报警（Alarm Code: E901）

**严重等级：** Critical（设备停机）

**现象描述：**
断路器跳闸或保险丝熔断，设备局部或全部断电。可能伴随焦糊气味或冒烟。

**处理步骤：**

1. 立即切断设备总电源
2. 确认无明火或烟雾，必要时使用灭火器（CO2灭火器，严禁使用水）
3. 检查跳闸的断路器回路编号
4. 检查对应回路的电气设备（电机、加热器、电源模块等）
5. 测量回路绝缘电阻（使用兆欧表，绝缘电阻应 > 1MΩ）
6. 排除故障后，合闸送电
7. 逐步恢复设备各子系统，观察电流是否正常
8. 记录故障回路、测量数据和修复措施

> ⚠️ 严禁在未排除故障前反复合闸。反复合闸可能扩大故障范围。

**预防措施：**
- 每月检查电气柜内接线端子紧固状态
- 每季度测量关键回路绝缘电阻
- 定期清洁电气柜内灰尘（使用干燥压缩空气）
- 确保电气柜散热风扇正常运行

---

## 5.3 非标准报警处理流程

对于未在标准列表中的报警，应执行以下流程：

1. 查阅设备操作手册（Manual），搜索对应报警代码
2. 在 DFMS 系统中检索历史故障案例
3. 根据相似案例进行处理
4. 若无法解决，联系设备厂商技术支持热线
5. 若厂商无法远程解决，安排厂商工程师现场服务
6. 所有非标准报警必须填写《异常报告单》并提交至设备经理

# 8. 安全规范

## 8.1 通用安全规则

- 禁止带电操作设备（除带电测量外，测量时需使用绝缘工具）
- 进入设备区域必须佩戴个人防护装备（PPE）：安全帽、防护眼镜、安全鞋
- 高风险操作（机器人示教、高压电气维修、液压系统维护）需两人以上共同执行
- 未经授权不得修改设备参数、PLC程序或安全回路设置
- 设备运行时不得将身体任何部位伸入运动区域

（有同事问"戴了手套能不能伸手进去取工件"，答案是**绝对不行**，手套反而可能被卷入更危险。——安全部 赵工）

## 8.2 LOTO（上锁挂牌）规程

以下情况必须执行LOTO程序：

- 设备机械维修（更换部件、调整机构）
- 电气系统维修（更换电气元件、修改接线）
- 液压/气动系统维修
- 进入机器人运动区域
- 任何需要旁路安全功能的操作

**LOTO执行步骤：**

1. 通知所有相关人员设备即将停机维修
2. 按下设备急停按钮
3. 切断设备总电源，关闭气源总阀、液压泵
4. 确认所有储能元件已释放能量（电容放电、气缸泄压、弹簧释能）
5. 使用个人专用锁具锁住电源开关
6. 悬挂"正在维修，禁止合闸"警示牌
7. 尝试启动设备，确认能源已隔离
8. 执行维修工作
9. 维修完成后，清点工具和人员
10. 取下警示牌，解除锁具
11. 恢复能源供应，试运行确认设备正常

## 8.3 个人防护装备（PPE）要求

|---|---|---|---|---|---|---|
| 日常操作 | ✅ | ✅ | ❌ | ✅ | 视噪音 | ❌ |
| 机械维修 | ✅ | ✅ | ✅ | ✅ | ✅ | ❌ |

# 9. 预防性维护计划

## 9.1 日检（Daily Check）

| 检查项目 | 标准值 | 方法 |
|---|---|---|
| 设备开机自检 | 无报警 | 观察HMI |
| 气源压力 | 0.5–0.7 MPa | 读取气压表 |
| 液压油油位 | 油标中线以上 | 目视检查 |
| 冷却液液位 | 标线以上 | 目视检查 |
| 设备运行声音 | 无异响 | 听觉检查 |
| 工作区域清洁 | 无杂物、无油污 | 目视检查 |
| 安全装置状态 | 正常无遮挡 | 功能测试 |

## 9.2 周检（Weekly Check）

- 清洁传感器感应面
- 检查气路接头紧固状态
- 检查机器人零点位置
- 清洁冷却散热器
- 检查液压系统管路接头
- 备份关键设备参数

## 9.3 月检（Monthly Check）

- 更换过滤减压阀滤芯
- 检查电气接线端子紧固状态
- 校准关键传感器
- 检查轴承润滑状态并补充润滑脂
- 检查机器人减速器油位
- 分析当月报警数据并提出改善建议

## 9.4 季检（Quarterly Check）

- 测量电气回路绝缘电阻
- 校验温度传感器精度
- 检查安全回路功能（急停、安全门、光栅）
- 检查液压油品质（粘度、水分、颗粒度）
- 备份PLC程序和参数
- 校验机器人TCP精度

## 9.5 年检（Annual Check）

- 全面检查设备机械精度
- 更换液压油和液压油滤芯
- 更换冷却液
- 检查设备基础和地脚螺栓
- 全面校准所有测量传感器
- 执行设备安全评估
- 更新设备维护档案

# 10. 应急预案

## 10.1 火灾应急

1. 立即按下急停按钮，切断设备电源
2. 使用CO2灭火器灭火（电气火灾严禁使用水或泡沫灭火器）
3. 如火势无法控制，立即拨打119并启动工厂消防预案
4. 通知所有人员沿安全通道撤离
5. 在安全地点集合清点人数

## 10.2 人员受伤应急

1. 立即按下急停按钮，确保设备停止
2. 对伤员进行初步急救（止血、固定）
3. 拨打120急救电话（重伤情况）
4. 保护现场，等待事故调查
5. 在2小时内提交《安全事故报告》

## 10.3 化学品泄漏应急

1. 切断泄漏源（关闭阀门）
2. 疏散泄漏区域人员
3. 穿戴防护装备进行清理
4. 大量泄漏时启动环保应急预案
5. 记录泄漏物质名称、数量和处理过程

# 13. 报警关闭与验收标准

为确保报警处理结果真实有效，避免“表面消警、实际隐患未消除”的情况，所有报警在关闭前必须满足以下验收要求。

## 13.1 报警关闭条件

满足以下全部条件后，方可在 DFMS 系统中关闭报警事件：

- 报警代码已从 HMI / SCADA / MES / DFMS 系统中消除
- 设备已恢复至正常待机或正常生产状态
- 已完成不少于 1 个完整运行循环验证
- 对 Critical 报警，须完成不少于 3 个完整运行循环验证
- 故障原因已初步明确，并形成处理记录
- 如涉及备件更换，已记录更换件名称、规格、数量和批次
- 如涉及安全功能触发，已完成安全功能复核
- 相关责任人已签字或在系统中完成电子确认

## 13.2 关闭前验证要求

### 13.2.1 一般报警验证要求

一般报警关闭前，应至少完成以下验证：

- 功能验证：设备功能恢复正常
- 状态验证：报警不再重复出现
- 参数验证：关键参数回到正常范围
- 记录验证：报警处理过程已完整录入系统

### 13.2.2 Critical 报警附加验证要求

对于 Critical 报警，除一般验证要求外，还需增加以下验证：

- 低速运行验证
- 空载运行验证
- 负载运行验证
- 连续运行稳定性验证
- 相关联设备联动验证
- 必要时由生产主管或设备经理确认后方可恢复批量生产

## 13.3 暂时关闭与正式关闭

如故障暂时缓解但根因尚未彻底消除，应将事件状态标记为：

- **临时恢复**：设备恢复运行，但需后续跟踪
- **待根因整改**：已恢复生产，但需安排后续维修
- **正式关闭**：故障已彻底处理完成，无需后续动作

> 禁止将“临时绕过”“降级运行”“待计划维修”的事件直接标记为正式关闭。

# 14. 根因分析与持续改进要求

对于重复性报警、高停机损失报警、涉及安全的报警，必须开展根因分析与改进闭环。

## 14.1 需开展根因分析的情形

出现以下任一情况时，必须启动 Root Cause Analysis（RCA）：

- 同一报警代码 7 天内重复出现 3 次及以上
- 单次停机时长超过 30 分钟
- 单次故障造成批量不良或报废风险
- 涉及人身安全、设备安全或消防风险
- 同类设备在不同工位重复发生类似故障
- 厂商已处理但问题反复出现

## 14.2 推荐分析方法

根据故障复杂程度，可采用以下方法之一或组合使用：

- 5 Why 分析法
- 鱼骨图（人、机、料、法、环、测）
- 故障树分析（FTA）
- Pareto 分析
- 时间序列趋势分析
- 相似历史案例对比分析

## 14.3 改进措施分类

根因分析后，改善措施应至少归类为以下一种：

- **临时遏制措施**：快速降低当前风险
- **永久纠正措施**：从根因层面消除问题
- **预防措施**：防止同类问题再次发生
- **标准化措施**：更新 SOP、点检表、培训材料
- **系统性措施**：更新 DFMS 规则、预警逻辑、备件策略

## 14.4 改进闭环要求

每项改进措施应包含以下内容：

- 改进事项名称
- 责任部门与责任人
- 目标完成日期
- 验证方式
- 验证结果
- 是否纳入标准文件修订

改进完成后，应由设备工程师验证效果，生产主管审核，必要时由设备经理批准关闭。

# 20. 设备报警处理操作规程（旧版V0.9草稿，仅供参考）

*说明：以下为2025年初编制SOP时的草稿版本，部分内容与正式版有差异。保留此处仅作为历史参考，实际执行请以正文（V1.0版）为准。*

**报警处理原则：**

第一条：所有报警必须先记录后处理，不得跳过记录步骤。
第二条：Critical报警必须由工程师确认后方可复位，操作员不得自行复位严重报警。
第三条：同一报警连续出现2次以上应上报工程师分析原因，不得反复复位。

**报警等级划分：**

一级报警（红色）：设备立即停止，需工程师处理
二级报警（黄色）：设备继续运行但需关注，操作员可处理
三级报警（蓝色）：仅提示信息，不影响生产

*（注：以上等级划分与正式版不同，正式版改为Critical/Warning/Info三级，颜色标识为红/黄/绿）*

**常见报警处理：**

气压不足（红色）：检查空压机是否运行，气压表是否正常，气管是否有漏气。处理时间约15-30分钟。

机器人碰撞（红色）：先切到手动模式，检查碰撞位置有没有卡住的东西，确认机械臂没有变形，然后回零复位。注意一定要先检查再复位，不能直接复位。

温度高（黄色）：看冷却风扇有没有转，冷却液够不够，如果不够就加。温度下来了就可以消警。

*（草稿结束，后续内容在正式版中已更新完善）*

# 22. 个人经验总结（非正式，仅供参考）

*以下为张工个人多年工作经验总结，不属于正式SOP内容，仅供学习参考。*

**关于气压不足报警（E101）的经验：**
- 约70%是滤芯堵塞，先查过滤减压阀能省时间
- 约20%是接头松动，用肥皂水快速定位漏点
- 约10%是其他原因（空压机故障、管路破裂、电磁阀漏气等）
- 空压机故障一般在夏季高温时多发，注意空压机房通风

**关于机器人碰撞报警（E203）的经验：**
- 大多数碰撞是工件定位偏移导致的，不是机器人本身问题
- 夹具定位销松动是常见原因，建议每天开工前用手晃一晃检查
- 如果碰撞后机器人某个轴声音变大，可能是减速器受损，需要厂商检查
- 轻微碰撞复位后建议空跑3个循环再上工件，确认轨迹正常

**关于驱动器故障（E402）的经验：**
- AL-01过载最常见，通常等10分钟冷却后重启就好
- AL-09位置偏差过大多半是机械卡阻，不是驱动器问题
- 夏天机房温度高时AL-12过热报警会增加，建议提前开空调
- 驱动器故障如果断电重启3次还不行，别再试了，直接上报或换备件

**关于通讯故障（E501）的经验：**
- 80%是网线问题（松动、损坏、端口故障），拔插和换端口是最快方法
- 如果是整条产线通讯中断，可能是交换机或PLC故障，级别更高
- 电磁干扰导致的通讯不稳定，通常是动力线和通讯线没分开走，需要整改
- 新设备上线时如果通讯报警多，检查IP地址有没有冲突
